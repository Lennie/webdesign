Unhosted

Freedom from web 2.0's monopoly platforms

Unhosted is a project for strengthening free software against hosted software. With our protocol, a website is only source code. Dynamic data is encrypted and decentralised, to per-user storage nodes. This benefits free software, as well as scalability, robustness, and online privacy.

Hacky Holidays Entry: Nathan Rugg's Scrapbook app

I'd like to enter the Hacky Holidays Competition. My entry is called Scrapbook. It's an HTML5 web app that lets you add images and text together to create an online scrapbook. It has two parts: an editor (located at scrapbook/admin/index.html) and a simple viewer (scrapbook/index.html). I used the webblog demo as a base for integrating it with Unhosted.

I've been working on this project for a little over a month (it began as a Christmas present for my wife). This is the first time I've used the <canvas> tag much, so that was my first hurdle. The biggest issues I had to overcome with the canvas tag was lack of layers or decent text processing (you can't calculate the height for some reason). I had to write all of the collision and text wrapping algorithms. Usually, I could figure out a formula pretty quickly, but sometimes I needed to get some help.

I heard about Unhosted on The H, and I thought that it might work well with my Scrapbook app, so I gave it a try. I was pleased to find how simple Unhosted was to integrate. It only took a few hours to get it working well. But I definitely hit some bumps and roadblocks along the way. My first problem was that I didn't realize that Unhosted needs referers turned on. I usually keep them off. But the main difficulty I'm facing is that processing large amounts of data just takes too long. Scrapbook works great with Unhosted if you only add small images or text. Medium sized images will work; however, I had to modify the database scheme. I had to change the `save` field to longtext instead of just text. Large images seem to make the browsers want to explode. Part of the problem is that currently some browsers can export canvas images as JPGs, but not all of them. So I have to use PNGs when saving, which can be a lot bigger. I would have liked for Scrapbook to save larger versions of the images, but right now, it only saves thumbnails. It would also help if Unhosted could set and get asyncronisly and provide a callback that could be triggered on success and on progress change so that I could show a progress bar when saving instead of locking the browser. I don't know if Unhosted uses web workers, but that could help too.

Overall, I really like Unhosted. Hopefully browsers will continue to speed up and make client-side encryption more practical when dealing with large amounts of data.
