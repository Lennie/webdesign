Unhosted

Freedom from web 2.0's monopoly platforms

Unhosted is a project for strengthening free software against hosted software. With our protocol, a website is only source code. Dynamic data is encrypted and decentralised, to per-user storage nodes. This benefits free software, as well as scalability, robustness, and online privacy.

Download the code at gitHub Read the manifesto

How does unhosted work?

A website is a specific web app, hosted on a specific server farm. There is a limited number of big centralized websites, that we all connect to. This is not how the web was intended to be.

Unhosted web apps are not hosted on one specific server farm. They use CORS to offload each user's encrypted data to where this specific user has specified in his WebFinger. The once-important server farms now become commodity infrastructure.

To find out more, check out our proof-of-concept (not real usable code yet, please contribute if you see errors in it), read our full manifesto, or ask a question on our mailing list.

We need you!

We need people to write unhosted web applications. We already got a few first demo apps out of our 'Hacky Holidays' competition, but we need more apps, and more complete ones. If you are able to write an unhosted web app, then you will be one of the first people in the world to have done that. If you don't know what unhosted is, read our manifesto first. We also have a wiki.

Get our tarball from github and play around with the 'hello world' examples. There is one Pub/Sub-model example and one messaging-model example. From there, take your inspiration and put a cool app together. If you get stuck with how to do something and want some advice, ask your questions on the mailing list and we'll answer them there. The third example is about cryptography and especially for the ubergeeks among us, but you won't need that one to win.

To make an unhosted web app work, you need to program all functionality client-side, and in a distributed architecture, so a bit of JavaScript skills come in handy, as does experience with scalability design. It's the perfect puzzle challenge for frontend developers, backend developers, sysadmins, and everyone inbetween! :)

WebDevs Show your cool apps!

Be the first person to write an unhosted WebApp! twitter, chat, farmville, poker, forum, webmail, whatever you can hack together with your web development and JavaScript skills.

You can use the hello-world examples from the tarball as a starting point. They include a Pub/Sub-model wappblog and a messaging-based wappmail example.

Geeks help with protocol improvements!

The UJ/0.1 protocol and the StarSign0.1 pki are very basic, and there are many things that can (and probably should) be added to them. Like support for federation, compatibility with CCN, improvement of security, designing a browser to browser discovery & friending mechanism, etcetera.
