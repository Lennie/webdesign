Unhosted

Freedom from web 2.0's monopoly platforms

Unhosted is an open web standard for decentralizing user data. On the unhosted web, data is stored per-user, under the user's control. That's where it belongs.

To find out more, check out our demo (log in with your unhosted account, or try 'demo@demo.redlibre.org', password 'demo'). You can also read our full manifesto, and our wiki, or ask a question on our mailing list.

How does it work?

In standard websites, user data is stored in the same server as the application.

On unhosted websites, data is taken away from app servers and lives on the data node you login with.

This is done using WebFinger, Auth2-cs, WebDAV, CORS and E2EE.

Benefits of being unhosted Five reasons to migrate now

The unhosted web architecture moves user data away from app servers. This has many advantages over the web2.0 architecture, both for users and for developers.

  • More privacy

    By letting unhosted web apps apply payload encryption to your data, you help prevent your unhosted account provider from infringing on your privacy.

  • More focus for the Web Developer

    The Web Developer can now focus on developing stunning applications without the worries of having to build a world-class infrastructure behind it. This makes it feasible for small scale web applications to compete with high-profile competitors.

  • Decentralization

    More robust/resilient architecture, with no Single Point of Failure - especially if combined with modern in-browser storage techniques.

  • Data freedom

    With your data living outside the app, you can switch back and forth between apps without even exporting and importing. Better than data liberation, this is data freedom brought to the web.

  • Scalability

    Developers only pay for seeding the statics until ISPs and browsers cache them, so a very basic http server can scale up to serve a very popular web app for many users.
